# OpenDDE 1.0.1

OpenDDE 1.0.1 turns the initial package publication into a hardened
cross-platform release. It keeps the public `opendde_v1` model and checkpoint
filenames unchanged while hardening installation, diagnostics, runtime
fallbacks, and release metadata.

## Highlights

- CPython 3.11, 3.12, and 3.13 packaging with explicit Linux and Apple Silicon
  macOS support boundaries; Windows remains unvalidated.
- Lightweight `opendde --help`, `opendde --version`, and `opendde doctor`
  startup before heavyweight inference imports.
- Explicit `--device auto|cpu|cuda` selection and safe PyTorch fallbacks for
  optional CUDA kernels.
- Validated four-GPU Fold-CP inference for larger inputs using native PyTorch
  triangle kernels.
- Standard CPU installation uses `opendde`; the Linux x86_64-only `opendde[gpu]`
  extra adds cuEquivariance acceleration, while the empty `opendde[cpu]` extra
  remains for compatibility with 1.0.0 install commands.
- Release-pinned checkpoint metadata for `opendde.pt` and
  `opendde_abag.pt`, including immutable source revision, sizes, and SHA-256
  digests.

## Upgrade

```bash
uv venv --python 3.11
uv pip install --python .venv --upgrade --torch-backend cpu \
  "opendde==1.0.1"
uv run --no-project --python .venv opendde --version
uv run --no-project --python .venv opendde doctor
```

For the validated Linux NVIDIA GPU configuration, replace the install command
with:

```bash
uv pip install --python .venv --upgrade --torch-backend cu126 \
  "opendde[gpu]==1.0.1"
```

The Docker image is versioned independently from the Python package:

```bash
docker pull aurekaresearch/opendde:v1
```
